My latest and greatest project:

When I began developing my data-conversion system, I wasn’t sure it could even be done. The challenge was enormous—automating a process long accepted as manual and tedious, under tight deadlines and with almost no resources. But through persistence, experimentation, and countless late nights, I built a hands-free solution that securely extracts client data from CDK and saves over $100K annually. More than a technical win, it replaced repetitive, unenjoyable work with something precise, auditable, and empowering. What began as uncertainty became proof that focus and creativity can transform both workflow and morale.

Here we have a video demonstration of this technology which I have developed in my free time. A quick summary of the process is: One - create a custom HttpClient which can directly interact with CDK’s API when it is given nothing more than the client’s username and password. Two - create HttpRequests with parameters that act as the CDK Drive functions (CDK Drive is the software that CDK runs on). Three - create and pull the files using the Url commands.

Performance‑Tuned SQL

I craft SQL that’s measurably faster—using temp tables and the right indexes where speed gains are on the table. Designed for clarity, reliability, and repeatable wins.

-- Stage raw → temp with narrow types
SELECT ...
INTO #stg
FROM dbo.Raw WITH (NOLOCK);

-- Add a helpful index to accelerate the heavy join
CREATE CLUSTERED INDEX IX_stg ON #stg (CustomerId, OrderDate);

-- Transform into the intake schema
INSERT dbo.Intake (CustomerId, OrderDate, ...)
SELECT s.CustomerId, s.OrderDate, ...
FROM #stg AS s
JOIN dbo.DimCustomer AS d
  ON d.CustomerId = s.CustomerId;
