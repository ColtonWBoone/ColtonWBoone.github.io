My latest and greatest project:

Ever thought of automating your entire data conversion process? That is, automating the data extraction, transferring, transforming and loading through a completely hands-free pipeline? Most likely your DMS needs to convert CDK's data through a labourious and costly process of making reports by hand, or by having the new client pay for a backup to be sent, with my newly created data extraction tool we could have the advantage of a low-cost, reliable, and maintainable data pipeline that runs through robust code.

Below is a demonstration of a technology which I have created in my free time. A quick summary of the process is: One: creates a custom HttpClient which can directly interact with CDK’s API when it is given nothing more than the client’s username and password. Two: create HttpRequests with parameters that mimic the CDK Drive functions (CDK Drive is the software that CDK runs on). Three: create and pull the files using the Url commands.

Basically, this technology allows someone to pull data from CDK's software as if it were a website, and gives full access to CDKs web information which would normally need many by-hand clicks to work through in CDK Drive.
